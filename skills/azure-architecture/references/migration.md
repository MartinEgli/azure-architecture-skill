# Azure Migration Reference

Use for `/azure migration`.

## Migration Lenses

- Current hosting model, runtime, data stores, integration, and operations
- Target Azure service options and rationale
- Rehost, replatform, refactor, retire, retain, or replace
- Identity and secrets changes
- Network access, private connectivity, DNS, and egress
- Data migration, cutover, rollback, and consistency
- Observability, backup, and incident readiness
- Cost, risk, and sequencing

## Handoffs

- Use `azure-cloud-migrate` for cross-cloud conversion workflows.
- Use `azure-prepare` for app-centric Azure deployment preparation.
- Use `azure-enterprise-infra-planner` for enterprise infrastructure IaC.
- Use `dotnet-engineering` for .NET modernization or code-level migration.
